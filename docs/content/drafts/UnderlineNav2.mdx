### Overflow Behaviour

#### Items without Icons

When overflow occurs, the component first hides icons if present to optimize for space and show as many items as possible.
If there is still overflow, the component will behave depending on the pointer. Please see below to see these behaviours.

```jsx live drafts
<UnderlineNav>
  <UnderlineNav.Item selected leadingIcon={CodeIcon}>
    Code
  </UnderlineNav.Item>
  <UnderlineNav.Item leadingIcon={IssueOpenedIcon} counter={30}>
    Issues
  </UnderlineNav.Item>
  <UnderlineNav.Item leadingIcon={GitPullRequestIcon} counter={3}>
    Pull requests
  </UnderlineNav.Item>
  <UnderlineNav.Item leadingIcon={CommentDiscussionIcon}>Discussions</UnderlineNav.Item>
  <UnderlineNav.Item leadingIcon={EyeIcon} counter={9}>
    Actions
  </UnderlineNav.Item>
  <UnderlineNav.Item leadingIcon={EyeIcon} counter={7}>
    Projects
  </UnderlineNav.Item>
  <UnderlineNav.Item leadingIcon={EyeIcon}>Security</UnderlineNav.Item>
  <UnderlineNav.Item leadingIcon={EyeIcon}>Insights</UnderlineNav.Item>
  <UnderlineNav.Item leadingIcon={EyeIcon} counter={1}>
    Settings
  </UnderlineNav.Item>
</UnderlineNav>
```

#### Display `More` menu for fine pointer devices

This overflow behaviour will occur on fine pointer devices that have fine pointers.

#### Scroll for coarse pointer devices

The scroll behaviour will occur on devices that have coarse pointers.

Depending on your device when viweing this example, you may see the scroll behaviour or the `More` menu behaviour.

```jsx live drafts
<UnderlineNav>
  <UnderlineNav.Item selected leadingIcon={CodeIcon}>

  </UnderlineNav.Item>

  <UnderlineNav.Item leadingIcon={EyeIcon} counter={9}>

  <UnderlineNav.Item leadingIcon={EyeIcon} counter={7}>
    Projects
```
